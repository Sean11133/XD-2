# spec.md — UI 大改版：深海水族風格 + 探索式檔案瀏覽

- **需求編號**：010
- **需求簡述**：UI-Redesign-Ocean
- **建立日期**：2025-01-30
- **版本**：v1.1（2026-04-01 決策更新）

---

## 1. 背景與目標

目前系統的 UI 為雙欄式佈局（左側樹狀 + 右側詳情），功能可用但體驗尚有優化空間。  
本次改版目標：以 VS Code Sidebar 與 Windows 檔案總管的熟悉互動模式為核心，融入輕量深海水族視覺風格，全面提升視覺一致性與操作效率。

---

## 2. 使用者故事

### US-01：VS Code 風格左側 Sidebar

**身為** 使用者，  
**我希望** 左側有一個類似 VS Code 的檔案樹，帶有工具列按鈕可快速新增資料夾、新增檔案，以及一鍵全部收合，  
**以便** 我能高效管理檔案結構而不需要分散注意力到其他面板。

**驗收標準：**

- [ ] Sidebar 頂部有三個圖示按鈕：+ 資料夾、+ 檔案、全部收合（Collapse All）
- [ ] 點擊「+ 資料夾」後，在目前選取的節點下方新增一個輸入框（inline 重命名模式），按 Enter 確認、Esc 取消
- [ ] 點擊「+ 檔案」後，同上邏輯，建立檔案並支援輸入副檔名（.txt / .jpg / .docx 等）
- [ ] 「全部收合」一鍵將樹狀全部折疊至根節點
- [ ] 樹狀節點展開/收合有輕微過渡動畫（150ms）
- [ ] 左側 Sidebar 支援拖曳調整寬度（最小 200px，最大 400px），拖曳把手（resize handle）位於 Sidebar 右邊緣
- [ ] 寬度值存入 `localStorage`，下次開啟自動恢復

---

### US-02：豐富標籤建立 UI

**身為** 使用者，  
**我希望** 建立標籤時有一個完整的 UI（顏色選擇、標籤名稱輸入、重要等級 1–5 選擇），  
**以便** 我的標籤能夠傳達更多語意資訊（不只是顏色，還有優先順序）。

**驗收標準：**

- [ ] 建立標籤的 UI 改為一個小型 Panel（可為右側浮動面板或下拉展開區塊）
- [ ] 提供顏色選擇器：預設至少 10 個顏色色票（深海主題配色為優先），支援自訂 HEX 輸入
- [ ] 提供標籤名稱輸入欄位（最長 20 字）
- [ ] 提供重要等級選擇（1–5 星），UI 使用星形圖示，選取時高亮顯示
- [ ] 完成建立後，標籤 Chip 上顯示：顏色圓點 + 名稱 + 星形徽章數量（如 ★★★）
- [ ] 編輯現有標籤支援相同 UI（點擊標籤 Chip 可進入編輯模式）

---

### US-03：Windows 檔案總管風格右側瀏覽區

**身為** 使用者，  
**我希望** 右側主區域是一個類似 Windows 檔案總管的資料夾內容視圖，  
**以便** 我能直觀地瀏覽、新增、刪除、複製節點，並支援上一步 / 下一步導覽。

**驗收標準：**

- [ ] 右側主區顯示目前所在資料夾的**直接子項目**（非全部子孫）
- [ ] 子項目以卡片格網（Grid）或清單（List）顯示，可切換
- [ ] 點擊資料夾卡片 → 進入該資料夾（更新導覽路徑）
- [ ] 右側頂部有**麵包屑導覽列（Breadcrumb）** + **上一步 / 下一步**按鈕
- [ ] 空白區域右鍵選單（可選）或工具列按鈕支援：新增資料夾、新增檔案、刪除、複製、貼上
- [ ] 點擊**檔案**卡片 → 右側滑入側邊抽屜（Side Drawer），顯示節點詳情（名稱、類型、標籤、屬性）
- [ ] 側邊抽屜寬度約 320px，從右側滑入（`transform: translateX`），可按 ✕ 或點擊遮罩關閉
- [ ] 點擊**資料夾**不觸發抽屜，直接進入該資料夾（導覽行為優先）
- [ ] 與左側樹狀同步：右側進入某資料夾時，左側樹自動展開到對應節點並高亮

---

### US-04：搜尋 + 篩選後匯出

**身為** 使用者，  
**我希望** 搜尋/篩選功能與匯出功能整合，讓我能只匯出目前篩選結果，  
**以便** 我可以按標籤、關鍵字篩選後做選擇性匯出，而非每次都匯出全部檔案樹。

**驗收標準：**

- [ ] 搜尋列與標籤篩選整合在同一個工具列區域（Search + Filter Bar）
- [ ] 匯出按鈕旁顯示目前篩選狀態指示（如「篩選結果：12 個節點」）
- [ ] 點擊「匯出」時，若有篩選條件 → 詢問「匯出篩選結果 or 全部」
- [ ] 匯出格式支援：XML（現有）、JSON（新增）、純文字清單（新增）
- [ ] 若篩選結果為空，匯出按鈕 disabled 並顯示提示

---

### US-05：深海水族輕度視覺主題

**身為** 使用者，  
**我希望** UI 整體色調有輕微的深海水族感，  
**以便** 日常使用時有沉浸式但不分心的視覺享受。

**驗收標準：**

- [ ] 主色調（Primary）改為深海藍（#0A2342 深色 / #1A4F7A 中）與青綠色（#00B4D8 / #48CAE4）
- [ ] 亮色主題：背景為淺海霧白（#F0F8FF / #E8F4FD），強調色為珊瑚橙（#FF8C69）
- [ ] 暗色主題：背景為深海夜（#0A1628 / #0D2137），邊框帶微微發光效果（`box-shadow: 0 0 8px rgba(0,180,216,0.3)`）
- [ ] Sidebar 與右側區域以淺色分割線分隔，加入 2–3 個海洋裝飾元素（SVG：氣泡、波紋 strip、小魚剪影）作為靜態點綴
- [ ] 裝飾元素僅作為 CSS 背景或 SVG 靜態圖，**不** 使用 JS 動畫（避免影響效能）
- [ ] `ocean` 作為**獨立第三主題**，現有 `light` / `dark` 主題完整保留
- [ ] **`ocean` 主題為系統預設主題**（`useTheme` 初始值改為 `'ocean'`，若 `localStorage` 無記錄則採用 `ocean`）
- [ ] 主題切換 UI 提供三個選項：☀️ 亮色 / 🌙 暗色 / 🌊 深海

## 3. 排除範圍（Out of Scope）

- 後端 API 持久化（新增/刪除節點僅更新前端狀態，不寫入任何後端儲存，待 011 需求處理）
- 拖曳排序（Drag & Drop）節點
- 多選操作（Multi-select）
- 國際化（i18n）
- 鍵盤快捷鍵（Keyboard shortcuts）

---

## 4. 技術限制與前提

- 技術棧：React 18 + TypeScript + Tailwind CSS v4 + Vite（不變）
- 現有 Domain Layer（`Directory`, `FileSystemNode`, `Label`, `FileSystemFacade`）維持不動，本需求只影響表示層與服務層
- 標籤 `Label` 模型需新增 `priority: number` (1–5) 欄位
- 匯出服務需新增 JSON 與純文字格式支援（擴充現有 `TemplateExporter` 架構）
- 深海主題以 CSS 自訂屬性（CSS Custom Properties）實作，整合至現有 `useTheme` hook
- `useTheme` 支援三個值：`'light' | 'dark' | 'ocean'`，預設值為 `'ocean'`
- 節點詳情顯示於右側 **Side Drawer**（滑入抽屜），不使用下方 Panel 或彈窗
- Sidebar 拖曳寬度使用 `mousedown/mousemove/mouseup` 原生事件或 `@dnd-kit/core`（視實作評估）

---

## 5. 非功能性需求

- **效能**：Sidebar 樹狀 100 個節點以內渲染不卡頓（< 16ms per frame）
- **響應式**：桌面端優先（1280px+），不需適配行動裝置
- **可維護性**：新增元件放在 `src/components/`，新增服務放在 `src/services/`，遵循現有命名慣例
- **無障礙（Accessibility）**：按鈕具備 `aria-label`，色彩對比度符合 WCAG AA
